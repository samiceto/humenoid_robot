# Grading Rubrics for Physical AI & Humanoid Robotics Course

## Assignment 1: Development Environment Setup and ROS 2 Basics
**Weight: 5%**

### Functionality (40 points)
- **Excellent (36-40 points)**: Code compiles without errors, all components function correctly, publisher/subscriber communication works flawlessly, custom message types implemented and tested
- **Good (32-35 points)**: Code compiles with minor warnings, most components work, communication mostly reliable, minor issues with custom messages
- **Satisfactory (28-31 points)**: Code compiles but with some issues, basic functionality works, communication has occasional problems
- **Needs Improvement (0-27 points)**: Code has compilation errors, functionality is limited or non-working

### Documentation (30 points)
- **Excellent (27-30 points)**: Comprehensive setup guide with troubleshooting tips, clear explanations, well-organized, includes screenshots where helpful
- **Good (24-26 points)**: Good documentation with most important information, some organization issues, clear explanations
- **Satisfactory (21-23 points)**: Basic documentation with essential information, some unclear sections
- **Needs Improvement (0-20 points)**: Incomplete or unclear documentation, missing important information

### Code Quality (20 points)
- **Excellent (18-20 points)**: Well-structured, properly commented, follows ROS 2 best practices, efficient implementation
- **Good (16-17 points)**: Good structure and comments, mostly follows best practices, minor improvements needed
- **Satisfactory (14-15 points)**: Basic structure, adequate comments, some best practices followed
- **Needs Improvement (0-13 points)**: Poor structure, insufficient comments, doesn't follow best practices

### Presentation (10 points)
- **Excellent (9-10 points)**: Clear, high-quality video demonstrating all functionality, good explanation of concepts
- **Good (8 points)**: Good video quality, demonstrates most functionality, adequate explanation
- **Satisfactory (7 points)**: Adequate video quality, demonstrates basic functionality
- **Needs Improvement (0-6 points)**: Poor video quality, doesn't demonstrate functionality, unclear explanation

## Overall Course Grading Scale

### Letter Grades
- **A+ (97-100%)**: Outstanding performance, exceptional understanding, innovative approaches
- **A (93-96%)**: Excellent performance, comprehensive understanding, sophisticated implementations
- **A- (90-92%)**: Very good performance, strong understanding, effective implementations
- **B+ (87-89%)**: Good performance, solid understanding, competent implementations
- **B (83-86%)**: Above average performance, adequate understanding, functional implementations
- **B- (80-82%)**: Average performance, basic understanding, basic implementations
- **C+ (77-79%)**: Below average performance, limited understanding, minimal implementations
- **C (73-76%)**: Poor performance, inadequate understanding, significant issues
- **C- (70-72%)**: Very poor performance, minimal understanding, major issues
- **F (0-69%)**: Unacceptable performance, no understanding, non-functional implementations

### Late Submission Penalties
- 1-2 days late: 10% penalty
- 3-7 days late: 25% penalty
- More than 7 days late: Not accepted without prior approval

### Academic Integrity
- Any form of plagiarism results in a zero grade for the assignment
- All work must be original and properly cited
- Collaboration must be documented
- Code from external sources must be attributed
