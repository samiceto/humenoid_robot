# Week 1-3 Syllabus: Physical AI & Humanoid Robotics Foundation

## Course Information
- **Course**: Physical AI & Humanoid Robotics: From Simulated Brains to Walking Bodies
- **Duration**: 3 weeks (Weeks 1-3 of 13-week course)
- **Prerequisites**: Basic programming knowledge (Python), mathematics (linear algebra basics)
- **Level**: Advanced undergraduate / Master's level
- **Credits**: 3 credit hours

## Learning Objectives

By the end of Week 3, students will be able to:
- Understand the fundamental concepts of Physical AI and humanoid robotics
- Explain the architecture and components of ROS 2
- Model robots using URDF and understand kinematic structures
- Install and configure the complete development environment
- Create basic ROS 2 nodes and message exchanges
- Design simple robot models with proper kinematic chains

## Week 1: Introduction to Physical AI and Humanoid Robotics

### Learning Objectives
- Define Physical AI and distinguish it from traditional AI
- Understand the evolution and current state of humanoid robotics
- Identify the sim-to-real challenges in robotics
- Navigate the course structure and technology stack

### Topics Covered
- Physical AI concepts and applications
- Historical development of humanoid robots
- Current state of humanoid robotics
- The sim-to-real challenge
- Course structure and learning path
- Technology stack overview (ROS 2, Isaac Sim, Isaac ROS, Jetson)

### Required Reading
- Chapter 1: Introduction to Physical AI and Humanoid Robotics
- ROS 2 documentation: Overview and concepts
- Isaac Sim documentation: Introduction

### Activities
- Complete Chapter 1 exercises
- Set up development environment following setup guides
- Explore Isaac Sim interface

### Assessment
- Quiz on Physical AI concepts (10% of grade)
- Environment setup verification (5% of grade)

### Time Allocation
- Reading: 4 hours
- Environment setup: 6 hours
- Exercises: 3 hours
- Quiz preparation: 1 hour

## Week 2: ROS 2 Fundamentals for Humanoid Systems

### Learning Objectives
- Install and configure ROS 2 Iron/Irwin
- Understand ROS 2 architecture and core concepts
- Create and run basic ROS 2 nodes
- Use topics, services, and actions for communication
- Apply ROS 2 tools for debugging and visualization

### Topics Covered
- ROS 2 installation and environment setup
- Nodes, topics, services, and actions
- Message types and communication patterns
- ROS 2 tools (ros2 run, ros2 topic, ros2 service, rqt)
- Creating a ROS 2 workspace
- ROS 2 for humanoid robotics applications

### Required Reading
- Chapter 2: ROS 2 Fundamentals for Humanoid Systems
- ROS 2 tutorials: Nodes and Topics
- ROS 2 documentation: Services and Actions

### Activities
- Complete Chapter 2 exercises
- Create custom ROS 2 nodes
- Implement topic-based communication
- Use ROS 2 tools for debugging

### Assessment
- ROS 2 node implementation (15% of grade)
- Topic communication demonstration (10% of grade)

### Time Allocation
- Reading: 3 hours
- Practical exercises: 8 hours
- Node implementation: 4 hours
- Testing and debugging: 2 hours

## Week 3: URDF and Robot Modeling

### Learning Objectives
- Understand URDF structure and components
- Create robot models with proper kinematic chains
- Define visual, collision, and inertial properties
- Validate URDF models and check kinematics
- Use Xacro for complex robot models

### Topics Covered
- URDF structure for humanoid robots
- Links and joints in humanoid robots
- Visual, collision, and inertial properties
- Joint types and limits
- Humanoid robot kinematic structure
- URDF validation and best practices
- Xacro macros and advanced features
- Isaac Sim integration considerations

### Required Reading
- Chapter 3: URDF and Robot Modeling
- URDF tutorials and specifications
- Xacro documentation

### Activities
- Complete Chapter 3 exercises
- Create a simple humanoid robot model
- Validate URDF model with check_urdf
- Implement Xacro macros for repeated elements

### Assessment
- URDF model creation (20% of grade)
- Kinematic structure validation (10% of grade)

### Time Allocation
- Reading: 3 hours
- Model creation: 6 hours
- Validation and testing: 3 hours
- Xacro implementation: 3 hours
